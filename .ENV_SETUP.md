**NEVER commit `.env.local` to git!**

Create these files in your project root:

### .env.local (Development)

```env
# Firebase Configuration
VITE_FIREBASE_API_KEY=your_actual_key
VITE_FIREBASE_AUTH_DOMAIN=your-project.firebaseapp.com
VITE_FIREBASE_PROJECT_ID=your-project-id
VITE_FIREBASE_STORAGE_BUCKET=your-bucket.appspot.com
VITE_FIREBASE_MESSAGING_SENDER_ID=your_sender_id
VITE_FIREBASE_APP_ID=your_app_id
VITE_FIREBASE_MEASUREMENT_ID=G-xxxx

# Admin Emails (comma-separated)
VITE_ADMIN_EMAILS=admin@danvion.com,sazid@danvion.com

# API Keys (if needed)
VITE_API_BASE_URL=http://localhost:5000

# Features (to enable/disable features)
VITE_ENABLE_ANALYTICS=true
VITE_ENABLE_ERROR_TRACKING=false
```

### .env.production (Production)

```env
VITE_FIREBASE_API_KEY=prod_key
VITE_FIREBASE_AUTH_DOMAIN=danvion-ltd.firebaseapp.com
# ... rest of production config
VITE_ENABLE_ERROR_TRACKING=true
```

### .env.staging (Staging)

```env
# Staging specific config
```

## .gitignore Protection

Make sure your `.gitignore` includes:

```
.env.local
.env.*.local
.env.production
.env.staging
.DS_Store
node_modules/
dist/
*.log
```

## Environment Validation

The app validates required env vars at runtime (see `src/config/environment.ts`)

## Firebase Security

- Store API keys in Firebase console only
- Never commit real credentials
- Rotate keys regularly
- Use Firebase App Check in production
